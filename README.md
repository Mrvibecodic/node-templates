# node-templates

Коллекция статических, готовых к деплою шаблонов сайтов — чистые
**HTML + CSS + JS**, без сборки и без зависимостей. Любой шаблон кладётся
на статический хостинг (Netlify, Vercel, nginx, GitHub Pages, S3) и сразу
работает.

## Шаблоны

| Шаблон | Описание |
| --- | --- |
| [`esports-stream-template`](./esports-stream-template) | ArenaCast — лендинг круглосуточной киберспортивной трансляции: вкладки с живыми стримами, фильтруемое расписание турниров, флоу закрытого членства и набор внутренних страниц. |

## Скрипт `uniquify-theme.sh`

Рандомизирует «отпечатки» темы (HTML / CSS / JS): делает из одной темы
несколько визуально идентичных, но технически уникальных копий. Структура
(DOM, селекторы, логика) и текстовый контент **не меняются** — меняются
только маркеры, по которым тему опознают краулеры/сканеры: имена кастомных
CSS-классов и id, имена CSS-переменных (`--var`), `data-*`-атрибуты,
сигнатурные комментарии и `<meta name="generator">`, а также размер файлов
и хеши (за счёт случайных невидимых вставок и перетасовки незначащих
пробелов).

Переименование работает по «белому списку»: вы сами перечисляете базовые
токены своей темы (например, префикс БЭМ-блоков), и заменяются только они —
сторонние классы (bootstrap, tailwind, иконочные шрифты) не ломаются.

**Зависимости:** bash 4+, coreutils (sed, awk, find, sort), perl 5 и один из
`md5sum | shasum | openssl`. Недостающие perl/утилиту хеширования скрипт
пытается доустановить сам (отключается через `--no-install-deps`).

### Использование

```bash
# Простой режим: положить скрипт в папку темы и запустить — он обработает
# html/css/js в этой же папке и соберёт результат в output.zip рядом.
./uniquify-theme.sh [опции]

# Расширенный режим: явно указать исходную папку/архив:
scripts/uniquify-theme.sh -s SRC_DIR [--zip OUT.zip | -o OUT_DIR] [опции]
```

С разными `--seed` вы получаете разные уникальные сборки; с одинаковым
`--seed` результат полностью воспроизводим.

### Опции

```
  -s, --src DIR          исходная папка темы (по умолчанию — папка скрипта)
  -o, --out DIR          папка для уникальной копии (по умолчанию — временная)
      --zip FILE         путь к итоговому zip (по умолчанию: <папка скрипта>/output.zip)
      --no-zip           не упаковывать результат в zip
      --seed STR         seed для воспроизводимости (по умолчанию — случайный)
      --prefix STR       префикс генерируемых имён (по умолчанию: u)
      --tokens "a,b,c"   список базовых классов/id для переименования
      --tokens-file F    файл со списком токенов (по одному на строку)
      --ext "html,css"   расширения для обработки (по умолчанию: html,htm,css,js)
      --no-vars          не переименовывать CSS-переменные (--var)
      --no-comments      не вставлять случайные комментарии-маркеры
      --no-jitter        не подмешивать незначащие пробелы
      --keep-generator   не трогать <meta name="generator">
      --no-install-deps  не доустанавливать недостающие зависимости автоматически
  -v, --verbose          подробный вывод
  -h, --help             справка
```

### Примеры

```bash
# Положить скрипт в папку темы и запустить — результат в output.zip рядом:
./uniquify-theme.sh --tokens "hero,card,btn,nav,footer"

# Явно указать исходную папку, итоговый zip и seed:
scripts/uniquify-theme.sh -s ./theme --zip ./build/site-01.zip \
    --tokens "hero,card,btn,nav,footer" --seed site-01
```
