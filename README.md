# node-templates

Коллекция статических, готовых к деплою шаблонов сайтов — чистые
**HTML + CSS + JS**, без сборки и без зависимостей. Любой шаблон кладётся
на статический хостинг (Netlify, Vercel, nginx, GitHub Pages, S3) и сразу
работает.

## Скрипт `uniquify-theme.sh`

Рандомизирует «отпечатки» темы (HTML / CSS / JS): делает из одной темы
несколько визуально идентичных, но технически уникальных копий. Структура
(DOM, селекторы, логика) и текстовый контент **не меняются** — меняются
только маркеры, по которым тему опознают краулеры/сканеры: имена кастомных
CSS-классов и id, имена CSS-переменных (`--var`), `data-*`-атрибуты,
сигнатурные комментарии и `<meta name="generator">`, а также размер файлов
и хеши (за счёт случайных невидимых вставок и перетасовки незначащих
пробелов).

### Авто-режим (по умолчанию)

Запускать можно **без аргументов** — скрипт сам собирает имена классов/id
из CSS-селекторов темы и имена CSS-переменных, строит карту переименования
и применяет её консистентно во всех html/css/js. Ручной список токенов
больше не нужен.

```bash
# Положить скрипт в папку темы и запустить — результат в output.zip рядом:
./uniquify-theme.sh

# Разные сборки: меняйте --seed (одинаковый seed = воспроизводимый результат):
./uniquify-theme.sh --seed site-01
./uniquify-theme.sh --seed site-02
```

Собираются **только** идентификаторы, реально объявленные в локальном CSS
(то есть «свои», а не сторонние bootstrap/tailwind). Имена-омонимы
HTML-тегов (`body`, `code`, `nav`, `form`, …) в авто-режиме исключаются,
чтобы не сломать JS-селекторы вроде `querySelector('nav')`. Картинки,
`url()`, текст и `@keyframes`-имена не затрагиваются.

### Полезные опции

```
  --tokens "a,b,c"    ручной белый список (отключает авто-сбор классов)
  --scan-html         добавить классы/id ещё и из HTML-атрибутов
  --exclude "a,b"     не переименовывать перечисленные имена
  --include-reserved  разрешить имена-омонимы тегов (осторожно с JS)
  --no-auto-tokens    выключить авто-сбор классов
  --no-vars           не трогать CSS-переменные
  --seed STR          seed для воспроизводимости
  --no-zip / -o DIR   отдать папкой вместо zip
  -v, --verbose       подробный лог;  -h, --help — полная справка
```

После сборки рядом с результатом пишется `.uniquify-manifest.txt` с seed и
полными картами переименования классов и переменных.

**Зависимости:** bash 4+, coreutils, perl 5 и один из `md5sum | shasum |
openssl`. Недостающее скрипт пытается доустановить сам (`--no-install-deps`
отключает).

## Конвенция: robots.txt во всех шаблонах

Каждый шаблон должен содержать в своём корне `robots.txt`, **полностью
запрещающий индексацию** всеми поисковиками и краулерами (включая AI-ботов:
GPTBot, CCBot, ClaudeBot, Google-Extended, PerplexityBot, Bytespider и т.д.):

```
User-agent: *
Disallow: /
```

`robots.txt` работает только в **корне домена**, куда задеплоен сайт
(`https://site.com/robots.txt`) — поэтому файл лежит в корне папки шаблона.
Готовый образец: [`esports-stream-template/robots.txt`](./esports-stream-template/robots.txt).
