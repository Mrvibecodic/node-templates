# WildPlanet — шаблон сайта «Зоопарки мира»

Лендинг-витрина живых трансляций из зоопарков и заповедников по всему
миру — с упором на обезьян и человекообразных. Анимированный глобус в
hero-секции («зоопарки мира»), сетка камер с ленивой загрузкой плееров,
фильтры по видам, переключатель «только приматы», блок-карта по
континентам, секция видов и форма обратной связи с валидацией.
Интерфейс на трёх языках: **English / Русский / 中文**. Вёрстка
адаптивная, мобильная версия оптимизирована (320 px и выше).

![Превью WildPlanet — десктоп и мобильная версия](../previews/worldzoo-stream-template.png)

## Стек

Чистые **HTML + CSS + JS** — без сборки и без зависимостей. Кладётся на
любой статический хостинг (Netlify, Vercel, nginx, GitHub Pages, S3) и
сразу работает.

## Структура

```
worldzoo-stream-template/
├── index.html                 # вся страница
├── robots.txt                 # полный запрет индексации
└── assets/
    ├── css/style.css          # светлая тема, анимации
    ├── js/
    │   ├── i18n.js            # переводы EN / RU / ZH
    │   ├── data.js           # список зоопарков и трансляций
    │   └── app.js            # рендер, фильтры, i18n, форма, глобус
    └── img/                   # локальные картинки животных + favicon
```

## Запуск

Откройте `index.html` в браузере или отдайте папку любым статическим
сервером:

```bash
npx serve .
```

> Часть камер (Twitch) требует, чтобы домен, на котором открыт сайт, был
> передан в параметре `parent`. Шаблон подставляет `location.hostname`
> автоматически, поэтому на `localhost` и на боевом домене Twitch-плееры
> работают без правок. По той же причине открывать страницу нужно через
> http(s), а не как `file://`.

## Как заменить название сайта

Название и подпись бренда вынесены в один объект в начале
`assets/js/app.js`:

```js
const SITE_CONFIG = {
  name: "WildPlanet",
  tagline: { en: "Zoos of the World", ru: "Зоопарки мира", zh: "世界动物园" },
  defaultLang: "en"
};
```

Поменяйте `name` — и новое имя автоматически подставится в шапку,
подвал, `<title>` и OG-теги (везде, где стоит атрибут `data-brand`).
`tagline` задаётся отдельно для каждого языка, `defaultLang` — язык по
умолчанию, если язык браузера не поддерживается.

## Языки

Все строки интерфейса лежат в `assets/js/i18n.js` (`en` / `ru` / `zh`).
Выбор языка сохраняется в `localStorage`; при первом заходе берётся язык
браузера. Чтобы добавить язык — продублируйте блок переводов с новым
кодом и добавьте кнопку `data-lang-btn="<код>"` в шапку.

## Как добавить или изменить камеру

Камеры описываются массивом `window.ZOO_CAMS` в `assets/js/data.js`.
Каждый объект:

```js
{
  id: "houston-gorilla",
  name: { en: "...", ru: "...", zh: "..." },
  city: { en: "...", ru: "...", zh: "..." },
  country: { en: "...", ru: "...", zh: "..." },
  region: "americas",          // americas | europe | asia | africa | oceania
  group: "apes",               // apes | monkeys | pandas | elephants | birds | savanna | other
  primate: true,               // попадает в фильтр «только приматы»
  species: { en: "...", ru: "...", zh: "..." },
  embed: "https://...",        // src для iframe плеера
  provider: "Ant Media",
  type: "iframe",              // "iframe" или "twitch"
  official: "https://...",     // ссылка на официальную страницу камеры
  image: "assets/img/houston-gorilla.jpg",
  live247: false               // true → бейдж «24/7», false → «дневные часы»
}
```

Для Twitch-камер укажите `type: "twitch"` и оставьте в `embed`
плейсхолдер `PARENT_HOST` — он будет заменён на текущий домен:
`https://player.twitch.tv/?channel=ИМЯ&parent=PARENT_HOST&muted=true&autoplay=true`.

## Источники трансляций

Все плееры встраиваются с официальных страниц соответствующих зоопарков
и сервисов (Camzone, CamStreamer, IPCamLive, HDOnTap, Ant Media, El Paso
Zoo, iPanda/CCTV, Twitch). Картинки животных скачаны локально из
Openverse (свободные лицензии). Список камер можно расширять — формат
описан выше.

## Форма обратной связи

Форма проверяет имя, e-mail и длину сообщения на стороне клиента и после
успешной валидации показывает подтверждение «Отправлено». Бэкенда у
формы нет: чтобы подключить реальную отправку, повесьте свой обработчик
на событие `submit` в `hookForm()` (`assets/js/app.js`) или укажите
`action` формы на свой эндпоинт. Контактные данные на сайте не
публикуются.

## Превью ссылок (OG-картинка)

Для GitHub Pages и превью ссылок используется чисто десктопный скриншот
1280×640: `previews/worldzoo-stream-template-desktop.png`, его копия в
шаблоне — `assets/img/og-image.png` (на неё указывает `og:image` в
`index.html`). Картинка для README — одна, десктоп + мобильная версия
вместе: `previews/worldzoo-stream-template.png`. Для задеплоенного сайта
замените `og:image` на абсолютный URL вашего домена — соцсети не
понимают относительные пути.

## Мобильная версия

Брейкпоинты: 900 px (одна колонка hero, бургер-меню) и 560 px
(компактный глобус, карточки и фильтры на всю ширину, карта 2×, секции
видов в столбец, скрытая CTA-кнопка в шапке). Сетка камер использует
`minmax(min(310px,100%),1fr)` и не переполняется даже на 320 px.
Анимации отключаются при `prefers-reduced-motion`.

## Создание уникальных копий темы

В корне репозитория лежит скрипт
[`uniquify-theme.sh`](../uniquify-theme.sh) — он делает из одной темы
несколько визуально идентичных, но технически уникальных копий:
консистентно переименовывает кастомные CSS-классы/id и переменные, меняет
data-маркеры, сигнатуры и хеши файлов, не трогая вёрстку и контент:

```bash
./uniquify-theme.sh

../uniquify-theme.sh -s worldzoo-stream-template -o worldzoo-copy
```

## robots.txt

В корне лежит `robots.txt`, **полностью запрещающий индексацию** всеми
поисковиками и AI-краулерами (GPTBot, CCBot, ClaudeBot, Google-Extended,
PerplexityBot, Bytespider и др.). `robots.txt` действует только в корне
домена, на который задеплоен сайт (`https://site.com/robots.txt`) —
держите его в корне при деплое.
